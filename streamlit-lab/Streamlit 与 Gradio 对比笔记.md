# **Streamlit 与 Gradio 对比笔记**

------

### 一、总体定位

- **Gradio**：专注于机器学习、大模型等交互式 Demo 的快速搭建。
   特点是上手快、自动生成输入输出界面、与 Hugging Face 集成紧密，特别适合展示模型效果。
- **Streamlit**：面向数据科学和科研计算可视化的 Web 应用框架。
   特点是灵活性强、支持复杂布局和可视化、适合构建分析工具和内部应用。后续虽加入了对大模型的支持，但功能深度不如 Gradio。

------

### 二、适用场景总结

| 目标                                     | 推荐               |
| ---------------------------------------- | ------------------ |
| 快速为模型制作可交互 Demo                | Gradio             |
| 构建带图表、控件、分析功能的完整 Web App | Streamlit          |
| 在 Hugging Face 上直接部署模型           | Gradio（集成度高） |
| 公司内部做数据展示或分析面板             | Streamlit          |

------

### 三、核心差异概述

| 对比项            | Streamlit                  | Gradio                 |
| ----------------- | -------------------------- | ---------------------- |
| 定位              | 通用数据与可视化应用       | 模型交互 Demo          |
| 开发方式          | Python 脚本式开发          | 定义输入输出组件       |
| UI 自定义         | 高，可通过 HTML/CSS 控制   | 较固定，简单统一       |
| 学习曲线          | 略高，但灵活可扩展         | 低，上手极快           |
| Hugging Face 集成 | 需要自行部署               | 原生支持               |
| 模型/LLM 支持     | 有，但生态偏弱             | 功能完善，支持主流模型 |
| 适合场景          | 数据分析、可视化、科研工具 | 模型演示、AI 应用原型  |

------

### 四、总结

- **Gradio** 更适合 **大模型、机器学习类项目的快速展示和部署**。
- **Streamlit** 更适合 **研究人员或工程师构建科学计算和数据可视化应用**。
- Streamlit 的灵活性带来了更高的自由度，虽然缺乏针对模型的深度集成，但适合扩展性和多样化场景。
